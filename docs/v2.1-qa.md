## Visual QA

### Global

- [ ] Page background uses ink palette (`--bg-0`), not flat `#000` + full-screen neon green body text.
- [ ] Primary UI font is **sans**; mono appears only on labels/chips/metadata.
- [ ] Text selection uses magenta-tinted highlight (not browser default only).
- [ ] Focus ring visible on Tab through: login email → password → submit → link.
- [ ] No wall of pure `#00FF00` on `#000` as the dominant look (legacy matrix terminal).

### Components (all four pages)

- [ ] **Button:** primary, secondary, ghost, danger visually distinct; disabled/loading states obvious.
- [ ] **Input:** label + error state styled; error not plain red unstyled text only.
- [ ] **Card:** auth and invites sections use consistent padding, border, radius.
- [ ] **Badge:** workspace role (admin/member) visible in planner header.
- [ ] **InlineAlert:** errors on auth/invites use component, not only `<p className="auth-error">`.

## Collaboration QA (two accounts)

**Setup:** User **A** (admin), User **B** (member). **A** invites **B**’s email; **B** registers with that email and accepts on `/invites`.

### Shared roster

- [ ] **A** sets display name in team/workspace panel → **B** sees the same name (after refresh).
- [ ] **B** sets own display name → **A** sees update.
- [ ] **B** cannot change **A**’s display name (UI disabled or RLS error with friendly message).
- [ ] **A** can change **B**’s display name (if admin-edit implemented; else document N/A).

### Assignee stability

- [ ] **A** creates task, assigns to **B** (by profile/user).
- [ ] **B** sees same assignee label/initials on that task.
- [ ] Hard refresh on both browsers — assignee unchanged.
- [ ] Supabase `tasks.assignee_user_id` = **B**’s `auth.users.id` (Table Editor spot check).

### localStorage regression

- [ ] DevTools → Application → Local Storage: **no** `teamMembers` key used for roster truth.
- [ ] New browser profile for **B** still sees roster from Supabase (not empty fake team).

## Security QA (RLS)

Use **member** account (**B**) unless noted.

- [ ] **B** cannot create sprint (UI hidden or disabled + API fails if forced).
- [ ] **B** cannot create/revoke invites on `/invites`.
- [ ] **B** **can** create/edit/delete **tasks** in shared workspace.
- [ ] **B** cannot `update` another user’s `workspace_profiles.display_name` via client (only own).
- [ ] Forced PostgREST call or tampered request shows **friendly** message (via `supabaseErrors.ts` where applicable), not raw policy jargon only.

**Admin (**A**):**

- [ ] Can create sprint, invite, revoke invite.
- [ ] Can CRUD tasks.
