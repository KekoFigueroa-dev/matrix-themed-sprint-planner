# V2.2 ship guide — what to do outside Cursor

Use this after **PR 5** is ready to push (branch `feat/pr5-polish-docs`). It covers everything you run yourself: Supabase, GitHub, Vercel, screenshots, and production sign-off.

**Production URL:** https://matrix-themed-sprint-planner.vercel.app  
**Full QA checklist:** [v2.2-qa.md](./v2.2-qa.md)

---

## Phase 1 — Push PR 5 and preview

### 1. Push the branch

In your own terminal (not required in Cursor):

```bash
cd /path/to/matrix-themed-sprint-planner
git checkout feat/pr5-polish-docs
git push -u origin feat/pr5-polish-docs
```

### 2. Open the pull request on GitHub

1. Go to https://github.com/KekoFigueroa-dev/matrix-themed-sprint-planner  
2. Open: **Compare & pull request** for `feat/pr5-polish-docs` → `main`  
   - Or: https://github.com/KekoFigueroa-dev/matrix-themed-sprint-planner/compare/main...feat/pr5-polish-docs?expand=1  
3. **Title:** `docs(polish): empty states, error copy, V2.2 QA + ship guide (PR5)`  
4. Paste the PR body from the agent or write: polish empty states, friendlier Supabase errors, `v2.2-qa.md`, `v2.2-ship-guide.md`, README screenshots section.  
5. Create the PR.

### 3. Wait for Vercel Preview

1. On the PR, find the **Vercel** bot comment with a **Preview** link (`*.vercel.app`).  
2. Click the preview URL.  
3. Quick smoke on the **preview** (not production yet):
   - `/login`, `/register` (red Register CTA)  
   - Sign in → planner empty states, add a task  
   - `/done`, `/invites`, `/about`  

If preview auth fails: Vercel → Project → **Settings → Environment Variables** — ensure `REACT_APP_SUPABASE_URL` and `REACT_APP_SUPABASE_PUBLISHABLE_KEY` (or anon key) are set for **Preview** as well as Production.

## Phase 3 — Screenshots (you capture, then commit)

The repo documents where images go; **you** take the PNGs on production (or preview after merge).

### 1. Capture (browser)

Use a logged-in admin account and a clean workspace if possible.

| File to save | Page | What to show |
|--------------|------|----------------|
| `docs/images/login.png` | `/login` | Matrix green auth card, red Register link |
| `docs/images/planner.png` | `/` | Project + sprint + tasks + stats |
| `docs/images/done.png` | `/done` | At least one done task or empty state |
| `docs/images/invites.png` | `/invites` | Pending or admin invite section |
| `docs/images/about.png` | `/about` | About page (optional but nice) |

**How to capture (example):**

1. Open the page at full desktop width (~1280px+).  
2. Browser dev tools → device toolbar off.  
3. Screenshot: Ubuntu `Shift+Print`, macOS `Cmd+Shift+4`, or browser extension.  
4. Save into `docs/images/` with the exact filenames above.

### 2. Commit screenshots (optional in PR 5 or right after merge)

```bash
git checkout main   # or stay on PR 5 branch before merge
git pull origin main
# copy PNGs into docs/images/
git add docs/images/*.png
git commit -m "docs: add V2.2 production screenshots"
git push origin main
```

If screenshots land after PR 5 merge, a tiny follow-up commit on `main` is fine.

## Phase 5 — Full production QA

Work through **[v2.2-qa.md](./v2.2-qa.md)** on **production**. Minimum path if short on time:

1. **Auth** — register (or login), hard refresh, still signed in.  
2. **Planner** — project switch, sprint, add task, all four statuses, mark **Done**, task leaves board.  
3. **Done** — task appears; **Restore** works; **Archive completed** + **Show archived** toggle.  
4. **Invites** — (two accounts) admin invites member email; member registers with that email, accepts on `/invites`.  
5. **Member security** — member cannot create project/sprint; can edit tasks.  
6. **Routes** — `/login`, `/invites`, `/done`, `/about` load (no Vercel 404).

Record pass/fail in the sign-off table at the bottom of `v2.2-qa.md`.

## Troubleshooting

| Symptom | What to check |
|---------|----------------|
| Blank page / config error | Vercel env vars; redeploy after fixing |
| CORS / network errors on login | Wrong `REACT_APP_SUPABASE_URL` (typo in hostname) |
| `Invalid API key` | Wrong publishable/anon key; Preview env not set |
| Task status update fails | Migration **#6** not applied |
| No projects in UI | Migration **#7** not applied; or admin needs to create project |
| RLS / permission errors | User role (admin vs member); read message in UI (should be friendly) |
| Invite not showing for invitee | Invited email must **exactly** match sign-in email; no email is sent by the app |
