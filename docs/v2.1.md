## Visual north star (mandatory — read before UI work)

### What you are building (one sentence)

A **dark, ink-purple product UI** that feels like a chill late-90s / early-web cyber lounge **reinterpreted with 2026 SaaS craft** — the way [stripe.dev](https://stripe.dev/) feels editorial, confident, and spacious, not noisy or retro for retro’s sake.

### Reference: what to steal from stripe.dev

| stripe.dev quality | Our translation |
|--------------------|-----------------|
| Clear typographic hierarchy (big titles, calm body, metadata subdued) | Sans for UI; mono only for labels, stats, “system” chips |
| Generous spacing and card-based sections | Planner = **panels/cards**, not boxed green borders on black |
| Dark surfaces with depth (layers, not flat `#000`) | `--bg-0` → `--bg-1` → `--bg-2` + soft shadow `--shadow-1` |
| Accent used sparingly for links, focus, primary CTA | Magenta `--accent-0` / purple `--accent-1` — **never** full-page neon text |
| Feels **responsive and alive** (layout breathes, hover/focus feedback) | CSS grid/flex that works on tablet; subtle transitions; Framer for list enter/exit only |
| Professional empty/loading/error states | Every route gets designed states via `Card` + `InlineAlert` + skeletons where needed |

### Reference: vaporwave / cyberpunk **chill** (mood, not meme)

**Do express:**

- Deep purple-black **ink** backgrounds with a **soft atmospheric gradient** (e.g. purple bloom top-center, very low opacity).
- **Thin grid or scanline texture** at 3–5% opacity (optional CSS background), not overpowering content.
- **Magenta / violet glow** on focus, primary buttons, active sprint — one glow language app-wide.
- **Mono “system” labels** (`code-label`) for metadata: workspace, sprint id snippet, “RLS”, role.
- Occasional **chrome/highlight** on key metrics (stats strip) — restrained.

**Do not express (old planner — explicitly retired in V2.1):**

- Matrix **#00FF00 on #000** as the main UI chrome.
- Monospace everywhere (body copy, buttons, headings).
- Dashed green borders, uppercase terminal headings, “hacker console” as the whole product.
- Neon soup, illegible contrast, gradient text behind paragraphs.

## Visual principles (strict)

- **Ink surfaces, clear hierarchy** — depth via `--bg-0` / `--bg-1` / `--bg-2`, not random gradients.
- **Magenta is accent, not body text** — default copy uses `--text-0` / `--text-1`; `--accent-0` for focus, primary actions, chips.
- **One glow rule** — subtle outer glow **only** on accent focus/active states (buttons, inputs). No glow on every border.
- **No gradients behind text** unless contrast is verified (WCAG-minded spot check).
- **Motion restrained** — one animation vocabulary (prefer existing Framer Motion patterns, shortened).

## Collaboration model (V2.1)

### New table: `workspace_profiles`

| Column | Type | Notes |
|--------|------|--------|
| `id` | uuid PK | `gen_random_uuid()` |
| `workspace_id` | uuid FK → `workspaces` | cascade delete |
| `user_id` | uuid FK → `auth.users` | cascade delete |
| `display_name` | text NOT NULL | default `''` |
| `created_at` | timestamptz | default `now()` |
| **Unique** | | `(workspace_id, user_id)` |

**RLS (intent):**

- **SELECT:** any member of `workspace_id`.
- **INSERT:** via `ensure_profile_for_user(workspace_id)` security definer RPC **or** narrow policy (own row when member).
- **UPDATE:** `user_id = auth.uid()` for own `display_name`; **admin** may update any row in workspace.

### Tasks: stable assignee

- Add `assignee_user_id uuid NULL` → `auth.users(id)` on delete set null.
- Stop writing `assignee_member_id` from UI (column may remain for migration safety; deprecate).
- Assignee picker options = `workspace_profiles` for active workspace (`display_name` + `user_id`).

### RPC: `ensure_profile_for_user(p_workspace_id uuid)`

- Security definer; verify `auth.uid()` is member via `workspace_members`.
- Upsert profile row if missing (display_name default: email local-part or empty).
- Called on planner load after `fetchActiveWorkspaceContext`.
- Backfill existing members in migration (one-time definer function).

### Frontend

| File | Role |
|------|------|
| `src/lib/teamDb.ts` | `fetchProfiles`, `updateMyDisplayName`, `adminUpdateDisplayName` |
| `src/pages/PlannerPage.tsx` | Remove `teamMembers` / `localStorage` |
| `src/components/TeamPanel.tsx` | Workspace members + display name edit |
| `src/lib/plannerDb.ts` | `assignee_user_id` read/write |

**Remove:** all `localStorage` keys `teamMembers` and assignee flows using `TeamMember.id: number` from `Date.now()`.
